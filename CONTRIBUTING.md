# Contributing to NanoPHP

Thank you for considering contributing to NanoPHP! Your help is essential in making this framework better for everyone.

## Contribution Guidelines

1. **Fork the Repository**: Create a fork of the repo and clone it to your local machine.
2. **Create a Branch**: Create a feature branch (`git checkout -b feature/amazing-feature`).
3. **Write Code**: Ensure your code follows PSR-12 coding standards.
4. **Test Your Changes**: Run existing tests and add new ones if necessary.
5. **Commit & Push**: Commit your changes with a clear message and push to your fork.
6. **Submit a Pull Request**: Open a PR to the `main` branch.

## Reporting Bugs

If you find a bug, please open an issue and include:

- A clear description of the bug.
- Steps to reproduce the issue.
- Your environment details (PHP version, OS, etc.).

## Code of Conduct

Please be respectful and helpful to others in the community. Let's build something great together!
